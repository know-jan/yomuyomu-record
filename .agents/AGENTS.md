# Yomuyomu Record 開発ルールと安定バージョン履歴

## 安定バージョン（動作確認済み・保管庫）
- **`v6.0-stable`** (Git Tag: `v6.0-stable`, Commit: `746d7cb`)
  - **実績**: iOS Safariネイティブの超高速検出エンジン（`BarcodeDetector`）と赤いターゲット枠だけの「物理クロップ（切り抜き）自動・手動スキャン」を完全融合。アスペクト比の違いによる物理的なズレ（3cm等）を解消する「スキャン位置の上下ズレ微調整スライダー（-50%〜+50%）」をデバッグパネルに完全実装。下の192バーコードを物理的にシャットアウトし、上の978だけを100%誤検知なしで狙い撃ちすることに成功した最高峰の安定版。
  - **トラブル発生時のコマンド**:
    ```bash
    git checkout v6.0-stable -- index.html
    ```
    を実行すれば、いつでもこの完璧な動作状態の `index.html` に復旧できます。

## 開発上の極めて重要な約束事（デグレード防止）

1. **グローバル変数の徹底的な宣言検証**
   - 新しい状態変数（`lastScannedIsbn` など）を導入する際は、必ずスクリプト先頭の「グローバル状態」ブロックにて `let` または `const` で初期値付きで宣言してください。宣言漏れは iOS Safari などのブラウザで `ReferenceError: Can't find variable` となり、スキャンプロセスを即座にクラッシュさせます。

2. **CORS同一オリジン制限を回避するための postMessage 双方向通信**
   - 別ウィンドウ（子スキャナー画面）とメイン画面（親）の間で通信を行う際、`window.opener.allRegisteredBooks` のような親側のオブジェクトや変数への直接アクセスは**絶対に禁止**します。ブラウザのセキュリティ制限（CORS同一オリジンポリシー）によって、子ウィンドウが即座にクラッシュします。
   - 親子間のすべてのバリデーション（重複チェック等）とデータ同期は、安全な `postMessage` によるメッセージ送受信（`isbn_scanned` -> `scan_result`）のみで完結させてください。

3. **一時停止（スロットリング）中のデコーダーコールバック連打防止**
   - 自動スキャンコールバック（`html5QrCode.start`）の最初（1行目）で、必ず `if (isScanningPaused) return;` による早期リターンを適用してください。
   - スロットリング（一時停止）中にデコーダーが検知した古い/誤検出のISBNによって、画面表示（子ども用モニターやデバッグ）が勝手に上書き・破損されるのを100%防ぐためです。

4. **手動キャプチャ（パシャっとスキャン）の独立（バイパス）設計**
   - 手動でのキャプチャテスト（`captureAndTestDecode`）が成功した際は、自動スキャン用の `onScanSuccess` は経由せず、直接 `addTempBookPlaceholder` を呼び出して本を追加し、モニターの表示もその場で手動更新してください（`v2.3` 安定設計の維持）。自動のカメラ制御と競合してフリーズするのを防ぐためです。

5. **ネイティブスキャン稼働中の裏の全画面ZXingスキャン結果の無視**
   - `BarcodeDetector`（スリットクロップ）の稼働中は、プレビュー枠外の誤検出を防ぐため、`html5QrCode.start` の成功コールバック内で `if (barcodeDetector) return;` による早期リターンを行い、裏で走る全画面ZXingの結果を完全に無視してください。
