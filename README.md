# 📚 よむよむレコード (Yomuyomu Record)

お子様向けの読書管理・図鑑風Webアプリケーションです。  
本のバーコードをカメラで読み取るだけで、書籍情報（タイトル、著者、カバー画像）を自動で検索し、読書履歴をスプレッドシートへ記録します。

---

## ✨ 主な機能

1. **バーコード連続スキャン (カメラ機能)**
   - スマホやタブレットのカメラで本のバーコード（ISBN）をスキャンするだけで、本の情報をインターネットから自動取得します。
   - 連続でどんどん読み込めるため、大量の本も一瞬でリスト化できます。
   - **iPad/iOS Safari対応**: Google Apps Script (GAS) のiframeサンドボックスによるカメラブロックを回避し、Netlify等の外部サーバーからのCORS通信を介したスキャンに対応しています。
2. **手動登録**
   - バーコードのない古い絵本や、手書きの本でもタイトルや著者、日付、感想、星評価（★1〜★5）を自分で書いて登録できます。
3. **読書貯金カウンター**
   - 読んだ本がたまると「コイン（🪙）」がカウントアップされ、お子様がゲーム感覚で楽しく読書を続けられます。
4. **よんだ本の一覧 (図鑑・本棚)**
   - 登録した本が綺麗な本のカバー画像とともに並びます。キーワード検索、所有区分（おうちの本・図書館の本）、ジャンルでの絞り込みが可能です。

---

## 🛠 システム構成

- **フロントエンド**: HTML5 / JavaScript (Vanilla JS) / Tailwind CSS (CDN) / FontAwesome
- **バーコードスキャンエンジン**: `html5-qrcode` ライブラリ
- **バックエンド/データベース**: Google Apps Script (GAS) + Googleスプレッドシート

---

## 🚀 セットアップ方法

### 1. GAS（Google Apps Script）の準備
1. Googleスプレッドシートを新規作成し、シート名を **「読書管理」** に変更します。
2. スプレッドシートのメニューから「拡張機能」 ＞ 「Apps Script」を開きます。
3. デフォルトのコードを削除し、このフォルダ内の `Code.js` の内容を貼り付けて保存します。
4. 右上の「デプロイ」 ＞ 「新しいデプロイ」をクリックします。
5. **「種類の選択」**で「ウェブアプリ」を選択します。
6. 設定を以下のように指定します：
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: 全員（または「全員（匿名）」）
7. デプロイを実行し、発行された **「ウェブアプリのURL」** ( `https://script.google.com/macros/s/.../exec` ) をコピーします。

### 2. フロントエンドの配置 (Netlify / GitHub Pages)
このリポジトリをGitHubにプッシュし、Netlifyと連携してホスティングします。

1. NetlifyやGitHub Pagesで公開されたURLを開きます。
2. 初回起動時に「システム設定」画面が表示されますので、上記ステップ1でコピーした **「GASウェブアプリのURL」** を貼り付けて保存します。
3. これで、スプレッドシートと外部フロントエンドが完全に通信できるようになります！
